#.🏎️ Project OVERCUT: T20 High-Leverage Strategy Engine Defending the Title: Data-Driven Decision Support for RCB’s 2026 Season

## 📌 Executive Summary: Standard cricket analytics focuses on historical averages. Project OVERCUT treats a T20 match as a high-speed "Race Strategy" problem. Inspired by F1 pit-wall dynamics and financial backtesting, this engine identifies Leverage Windows—specific 6-ball intervals where a strategic intervention (an Impact Sub or a bowling change) yields the highest marginal increase in Win Probability.

🚀 Key Innovation: The "F1 Pit-Wall" FrameworkMost T20 dashboards are static. This engine is dynamic, focusing on three "unique" metrics: Leverage Index ($LI$): A volatility metric that identifies "High-Stress" balls where the game is actually won or lost. Exploitation Windows: Identifying the exact point of "Bowler Predictability" (the T20 version of tire degradation) where a batter's Strike Rate is statistically likely to spike.Impact-Sub Optimization: A predictive model for the 2026 roster (featuring Bethell, V. Iyer, and Kohli) to determine the optimal entry point for the "Impact Player" to maximize Net Run Rate (NRR).

🛠️ Tech Stack: Data Processing: Python (Pandas) for cleaning 2025/2026 ball-by-ball datasets.Backtesting Engine: Custom Python scripts to test 2025 match-decisions against "Optimal" model suggestions.Visualization: Power BI (Dark Luxury Theme) utilizing advanced DAX for real-time Win Probability swings.

📊 Dashboard Features: The Race Map: A dual-axis visual tracking Win Prob % vs. Leverage Index.The Pit Window: A gauge visual indicating the "Green Zone" for an Impact Substitution.Matchup Matrix: A heat map of RCB’s 2026 squad vs. the specific bowling archetypes of the 2026 opposition (e.g., Left-arm Wrist Spin).
